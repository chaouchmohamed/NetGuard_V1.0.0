## рҹ“– Overview

**NetGuard** is a production-ready network attack detection system that uses **Isolation Forest** machine learning to identify anomalous network traffic patterns in real-time. It features a cyberpunk-themed dashboard with live traffic visualization, threat alerts, anomaly scoring charts, and a network flow map вҖ” all updating live via WebSocket.

No root privileges required. No real network access needed. Fully synthetic traffic simulation built-in.

---

## вңЁ Key Features

| Feature | Description |
|---------|-------------|
| рҹӨ– **ML Detection** | Isolation Forest model trained on 10,000 synthetic normal traffic samples |
| вҡЎ **Real-time Stream** | WebSocket-based live feed updating every 500ms |
| рҹҺҜ **5 Attack Types** | DDoS, Port Scan, SYN Flood, Brute Force, Data Exfiltration |
| рҹ“Ҡ **Live Dashboard** | Anomaly chart, attack distribution donut, network flow map |
| рҹ”” **Severity Alerts** | 4-tier classification: LOW / MEDIUM / HIGH / CRITICAL |
| рҹ’ҫ **Persistent Storage** | SQLite database for full alert history and traffic logs |
| рҹ”„ **Auto-train** | Model trains automatically on first run if no `.pkl` exists |
| рҹҺ® **Attack Simulator** | Built-in script to generate real test attacks |
| рҹҗі **Docker Ready** | One-command deployment with Docker Compose |

## рҹҡҖ Quick Start

### Prerequisites

- Python **3.8+**
- Node.js **16+**
- npm or yarn

### 1. Clone the repository

```bash
git clone https://github.com/chaouchmohamed/NetGuard_V1.0.0.git
cd NetGuard_V1.0.0
```

### 2. Run setup script

**Linux / macOS:**
```bash
chmod +x scripts/setup.sh
./scripts/setup.sh
```

**Windows:**
```batch
scripts\setup.bat
```

This will:
- Create a Python virtual environment
- Install all Python dependencies
- Train the ML model (`detector.pkl`)
- Install all Node.js packages

### 3. Start the application

**Linux / macOS:**
```bash
./scripts/run_dev.sh
```

**Windows:**
```batch
scripts\run_dev.bat
```

### 4. Open the dashboard

| Service | URL |
|---------|-----|
| рҹ–ҘпёҸ Frontend Dashboard | http://localhost:5173 |
| вҡҷпёҸ Backend API | http://localhost:8000 |
| рҹ“ҡ API Docs (Swagger) | http://localhost:8000/docs |

## рҹ“Ҡ Detected Attack Types

| Attack | Icon | Description | Severity |
|--------|------|-------------|----------|
| **DDoS** | рҹҢҠ | High-volume traffic from many source IPs targeting one destination | рҹ”ҙ CRITICAL |
| **Port Scan** | рҹ”Қ | Sequential port probing with SYN-only packets | рҹҹ  HIGH |
| **SYN Flood** | вҡЎ | Rapid TCP SYN packets with no ACK responses | рҹҹ  HIGH |
| **Brute Force** | рҹ”Ё | Repeated connection attempts to SSH (22) or RDP (3389) | рҹҹЎ MEDIUM |

## рҹ§  How It Works

### Isolation Forest Algorithm

Isolation Forest detects anomalies by randomly isolating observations:

1. **Random partitioning** вҖ” Randomly select a feature and a split value
2. **Isolation depth** вҖ” Anomalies are isolated in fewer splits (shorter tree paths)
3. **Score normalization** вҖ” Path length is normalized to a 0вҖ“1 anomaly score

```
Normal traffic  вҶ’ deep in tree   вҶ’ low score  (< 0.5) вң“
Attack traffic  вҶ’ near tree root вҶ’ high score (> 0.5) вҡ 
```

### The 13 Features Used

| # | Feature | Description |
|---|---------|-------------|
| 1 | `packet_size` | Bytes per packet |
| 2 | `inter_arrival_time` | Seconds between packets |
| 3 | `port_number` | Destination port |
| 4 | `protocol_encoded` | TCP=0, UDP=1, ICMP=2 |
| 5 | `packets_per_second` | Current PPS rate |
| 6 | `bytes_per_second` | Current throughput |
| 7 | `src_port` | Source port |
| 8 | `dst_port` | Destination port |
| 9 | `flag_syn` | TCP SYN flag (0/1) |
| 10 | `flag_ack` | TCP ACK flag (0/1) |
| 11 | `flag_fin` | TCP FIN flag (0/1) |
| 12 | `flag_rst` | TCP RST flag (0/1) |
| 13 | `connection_duration` | Session length in seconds |

### Severity Classification

| Score Range | Severity | Color |
|-------------|----------|-------|
| 0.85 вҖ“ 1.00 | CRITICAL | рҹ”ҙ |
| 0.65 вҖ“ 0.85 | HIGH | рҹҹ  |
| 0.45 вҖ“ 0.65 | MEDIUM | рҹҹЎ |
| 0.00 вҖ“ 0.45 | LOW | рҹ”ө |

Attack type also elevates severity вҖ” e.g. `data_exfiltration` is always at least CRITICAL.
